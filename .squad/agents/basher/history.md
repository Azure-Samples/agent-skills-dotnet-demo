- **Owner:** Bruno Capuano
- **Project:** Agent Skills .NET Demo — A .NET 10 console app showcasing Microsoft Agent Framework Agent Skills with Azure OpenAI. Three demo skills: prompt-only (meeting-notes), .NET script (code-reviewer), Python script (data-analyzer).
- **Stack:** .NET 10, C#, Python, Azure OpenAI, Bicep, PowerShell, Azure Developer CLI (azd)
- **Created:** 2026-04-08

## Learnings

- **Infrastructure scan (2026-04-08):** Project uses Bicep for Azure OpenAI deployment. Bicep is modern (2024-10-01 API versions), modular (cognitive-services + model-deployment), but missing several production best practices: no parameterization for capacity/SKU, hardcoded deployment names and versions, missing lifecycle/diagnostics outputs, no encryption/network isolation configs, and zero monitoring/alerting infrastructure. azd integration is minimal (only ENDPOINT and DEPLOYMENT_NAME outputs). Deployment scripts are PowerShell with solid error handling but lack idempotency checks and dry-run capability. No CI/CD for IaC validation (Bicep lint, ARM template validation, cost estimation). CI/CD workflows exist but are Squad-specific (triage/label sync/heartbeat) — no build, test, or deploy pipelines. Security posture acceptable (no hardcoded secrets, uses dotnet user-secrets correctly), but public network access enabled on OpenAI account with no vnet/network restrictions. azure.yaml minimal but compliant. Missing opportunity: parameterized deployments for dev/test/prod environments.
- **Cross-Domain Integration (2026-04-09):** Architecture scenarios (multi-agent orchestration, streaming, multi-turn conversation) will impact infrastructure patterns. HIGH-priority multi-turn feature requires reliable session/conversation context storage — design should consider Azure persistence layer or session backend. Streaming with token tracking needs observability setup (Application Insights/Log Analytics) — currently missing from Bicep. Multi-agent scenario suggests scalability patterns (AKS, Container Apps, load balancing) beyond current single-deployment scope. Quality gaps (zero test coverage, unhandled errors) must be addressed before infrastructure patterns are finalized; CI/CD validation pipelines needed for confidence in parameterized deployments.
- **Phase 1 Infra Improvements (2026-04-09):** Completed all 4 tasks: (1) setup.ps1 now idempotent with -WhatIf, -Force, -SkipSecrets; checks existing secrets before overwriting; only sets secrets after deployment verification; pre-flight checks for az/python.
